# Отчёт по проекту

## 1. Описание

Проект реализует контейнеризацию  `booking-service`. с помощью Docker и Kubernetes с использованием Helm. 

Реализованы проверки состояния сервиса (health checks) и тестирование DNS внутри кластера для проверки его доступности.

---

## 2. Docker

- Базовый образ: `python:3.11-slim`
- Healthcheck с использованием `curl` на `/ping`
- Запуск приложения FastAPI через Uvicorn

---

## 3. Развёртывание через Helm

### Конфигурация Deployment

- Количество реплик настраивается через `values.yaml`.
- Переменные окружения передаются через Helm.
- Настроены `livenessProbe` и `readinessProbe`:
  - Liveness: `/ping`
  - Readiness: `/ready`
- Ресурсы (CPU, RAM) определяются через Helm values.

### Конфигурация Service

- Тип сервиса: `ClusterIP`
- Открывает порт `80`, перенаправляя на порт контейнера `8080`.

---

## 4. Тестирование в Kubernetes

### Проверка DNS и доступности сервиса

- Настроен скрипт `check-dns.sh` для проверки внутреннего DNS.
- Шаги скрипта:
  1. Проверка существования подов с меткой `app=booking-service`.
  2. Ожидание готовности подов.
  3. Запуск временного pod с `curl` для обращения к `http://booking-service:80/ping`.
- Результат: сервис вернул `{"status":"ok"}`, что подтверждает доступность сервиса.
